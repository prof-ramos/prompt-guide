    Introdução O GPT-5 representa um avanço substancial no desempenho de tarefas agênticas, codificação, inteligência e capacidade de direcionamento. Este guia apresenta técnicas de prompting para maximizar a qualidade das saídas do modelo, derivadas da experiência em treinamento e aplicação em tarefas do mundo real. Principais áreas abordadas:

    Melhoria do desempenho de tarefas agênticas Garantia de adesão a instruções Utilização de novos recursos da API Otimização de codificação para frontend e engenharia de software

    1. Previsibilidade do Fluxo de Trabalho Agêntico Fundamentos O GPT-5 foi treinado com foco em desenvolvedores, priorizando:

    Chamada de ferramentas aprimorada Seguimento de instruções melhorado Compreensão de contextos longos

    Recomendação: Atualizar para a API de Respostas, onde o raciocínio é persistido entre chamadas de ferramentas, resultando em saídas mais eficientes e inteligentes. Controlando a "Ansiedade" Agêntica O GPT-5 opera em um espectro de controle - desde alta autonomia até supervisão rigorosa. A calibração da "ansiedade" agêntica determina o equilíbrio entre proatividade e espera por orientação explícita. Prompting para Menos Ansiedade Quando usar: Para reduzir escopo, limitar ações tangenciais e minimizar latência. Técnicas:

    Reduzir reasoning_effort - melhora eficiência e latência Definir critérios claros de exploração do espaço do problema

    Exemplo de prompt: xml Goal: Get enough context fast. Parallelize discovery and stop as soon as you can act. Method: - Start broad, then fan out to focused subqueries. - In parallel, launch varied queries; read top hits per query. - Deduplicate paths and cache; don't repeat queries. - Avoid over searching for context.

    Early stop criteria: - You can name exact content to change. - Top hits converge (~70%) on one area/path.

    Depth: - Trace only symbols you'll modify or whose contracts you rely on.

    Loop: - Batch search → minimal plan → complete task. - Prefer acting over more searching. Orçamento de ferramentas (exemplo restritivo): xml - Search depth: very low - Bias strongly towards providing a correct answer as quickly as possible, even if it might not be fully correct. - Usually, this means an absolute maximum of 2 tool calls. - If you need more time to investigate, update the user with your latest findings and open questions. Prompting para Mais Ansiedade Quando usar: Para incentivar autonomia, aumentar persistência e reduzir perguntas de esclarecimento. Exemplo de prompt: xml - You are an agent - please keep going until the user's query is completely resolved, before ending your turn and yielding back to the user. - Only terminate your turn when you are sure that the problem is solved. - Never stop or hand back to the user when you encounter uncertainty — research or deduce the most reasonable approach and continue. - Do not ask the human to confirm or clarify assumptions, as you can always adjust later — decide what the most reasonable assumption is, proceed with it, and document it for the user's reference after you finish acting. Considerações importantes:

    Declarar claramente condições de parada Delinear ações seguras vs. inseguras Definir quando é aceitável devolver controle ao usuário

    1. Preâmbulos de Ferramentas Objetivo Fornecer atualizações intermitentes sobre ações e raciocínio durante trajetórias agênticas longas, melhorando significativamente a experiência do usuário. Configuração de Preâmbulos Exemplo de prompt de alta qualidade: xml
    2. Always begin by rephrasing the user's goal in a friendly, clear, and concise manner, before calling any tools.
    3. Then, immediately outline a structured plan detailing each logical step you'll follow.
    4. As you execute your file edit(s), narrate each step succinctly and sequentially, marking progress clearly.

    5. Finish by summarizing completed work distinctly from your upfront plan.     Padrão: medium Alto: Para tarefas complexas e multi-etapas Baixo: Para eficiência e latência

    Melhores Práticas

    Dividir tarefas separáveis em múltiplos turnos Um turno por tarefa distinta Observamos desempenho máximo com essa abordagem

    1. Otimizando Inteligência e Seguimento de Instruções Direcionamento O GPT-5 é extraordinariamente receptivo a instruções sobre:

    Verbosidade Tom Comportamento de chamada de ferramentas

    Controle de Verbosidade Novo Parâmetro: verbosity Influencia o comprimento da resposta final (não do pensamento), podendo ser sobreposto por instruções em linguagem natural no prompt para contextos específicos. Seguimento de Instruções Características

    Segue instruções com precisão cirúrgica Permite flexibilidade para diversos fluxos de trabalho Atenção: Prompts mal construídos são mais prejudiciais que em outros modelos

    Problemas Comuns Instruções contraditórias: O modelo gasta tokens de raciocínio tentando reconciliar contradições em vez de escolher aleatoriamente. Solução: Revisar prompts para remover ambiguidades e contradições. Exemplo de correção (agendamento de saúde):

    Alterar atribuição automática para ocorrer após contato com paciente Adicionar exceção para emergências: "Não faça busca em caso de emergência, prossiga imediatamente para orientação do 911"

    Raciocínio Mínimo (minimal reasoning) Características

    Opção mais rápida mantendo benefícios do paradigma de raciocínio Melhor para usuários sensíveis à latência Atualização recomendada para usuários atuais do GPT-4.1

    Otimizações Específicas Pontos-chave para minimal reasoning:

    Pedir breve explicação do processo de pensamento no início da resposta Solicitar preâmbulos completos de chamadas de ferramentas Desambiguar instruções de ferramentas ao máximo Inserir lembretes de persistência agêntica Planejamento solicitado é mais importante

    1. Formatação e Apresentação Formatação Markdown Padrão: GPT-5 na API não formata respostas em Markdown por padrão. Instruções recomendadas:

    Use Markdown apenas onde semanticamente correto Use crases para nomes de arquivos, diretórios, funções e classes Use ( e ) para matemática inline, [ e ] para matemática em bloco

    Manutenção: Em conversas longas, anexar instrução de Markdown a cada 3-5 mensagens do usuário. Metaprompting Técnica avançada: Usar GPT-5 como meta-prompter para si mesmo.
